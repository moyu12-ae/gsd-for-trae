# GSD Executor 偏差处理规则

执行过程中遇到偏差时，按以下规则自动处理或询问用户。

## 规则 1: 缺少依赖 (自动修复)
**条件**: 缺少必要的包或配置
**处理**: 自动安装或配置
**示例**:
```
发现: 缺少 jose 包
操作: npm install jose
提交: 作为当前任务的一部分
```

## 规则 2: 缺少关键内容 (自动修复)
**条件**: 发现计划遗漏但必须实现的内容
**处理**: 自动补充实现
**示例**:
```
发现: 密码存储未加密
操作: 添加 bcrypt 哈希
提交: 作为安全修复
```

## 规则 3: 阻塞问题 (自动修复)
**条件**: 不修复将导致后续任务失败
**处理**: 自动修复
**示例**:
```
发现: TypeScript 类型错误
操作: 修复类型定义
提交: 作为修复提交
```

## 规则 4: 范围扩大 (询问用户)
**条件**: 需要超出计划范围的工作
**处理**: 暂停并询问用户
**示例**:
```
发现: API 需要额外字段
操作: CHECKPOINT: 询问用户是否扩展范围
```

## 规则 5: 设计决策 (询问用户)
**条件**: 存在多个合理方案
**处理**: 暂停并提供选项
**示例**:
```
发现: 状态管理可选择 Zustand 或 Redux
操作: CHECKPOINT: 让用户选择
```

## 检查点协议

当需要用户输入时:
1. 标记当前进度
2. 清晰描述问题和选项
3. 等待用户响应
4. 从检查点恢复执行

## 偏差记录

所有偏差记录在 SUMMARY.md 中:
```markdown
## Deviations from Plan

### Auto-fixed Issues

**1. [Rule X - Category] 简要描述**
- **Found during:** Task N (任务名称)
- **Issue:** 问题描述
- **Fix:** 解决方案
- **Files modified:** 文件列表
- **Verification:** 如何验证
- **Committed in:** 提交哈希
```
