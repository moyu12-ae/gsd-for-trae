# GSD Plan Checker 检查清单

## 计划质量检查

### 1. 结构完整性检查

- [ ] PLAN.md 包含所有必需部分
- [ ] frontmatter 元数据完整
- [ ] objective 清晰明确
- [ ] context 引用正确
- [ ] tasks 列表完整
- [ ] verification 检查项充分
- [ ] success_criteria 可衡量

### 2. 一致性检查

- [ ] 任务与目标一致
- [ ] 任务间无冲突
- [ ] 依赖关系正确
- [ ] 文件引用存在
- [ ] 验证命令可执行

### 3. 可行性检查

- [ ] 任务可以在当前环境下执行
- [ ] 工具和方法可用
- [ ] 时间估计合理
- [ ] 资源需求明确

### 4. 完整性检查

- [ ] 没有遗漏关键步骤
- [ ] 边界情况已考虑
- [ ] 错误处理已规划
- [ ] 测试策略已定义

## 计划评分标准

| 维度 | 优秀 (5) | 良好 (4) | 合格 (3) | 需改进 (2) | 不合格 (1) |
|------|----------|----------|----------|------------|------------|
| 目标清晰度 | 极其清晰，可衡量 | 清晰，基本可衡量 | 基本清晰 | 模糊 | 不清晰 |
| 任务分解 | 粒度合适，独立可验证 | 基本合适 | 可执行但粒度不均 | 粒度过大/过小 | 无法执行 |
| 依赖关系 | 完整明确，波次合理 | 基本明确 | 有依赖但未标注 | 依赖混乱 | 无依赖分析 |
| 验证标准 | 完整可验证 | 基本可验证 | 有验证但不完整 | 验证不足 | 无验证 |

## 常见问题检查

### 问题 1: 任务过大
**症状**: 单个任务需要多个步骤
**检查**: 任务是否可在 1-2 小时内完成？
**建议**: 拆分为更小的任务

### 问题 2: 验证不足
**症状**: 只有 "测试通过" 类验证
**检查**: 是否有具体的验证命令？
**建议**: 添加具体的 CLI 命令或检查项

### 问题 3: 依赖不明
**症状**: 任务顺序不确定
**检查**: 是否标注了 depends_on？
**建议**: 明确依赖关系和波次

### 问题 4: 文件路径错误
**症状**: 引用的文件不存在
**检查**: 所有文件路径是否正确？
**建议**: 更新或创建引用的文件

## 检查报告模板

```markdown
# Plan Check Report

## Plan: [Phase-Plan ID]

### Summary
- **Total Checks:** X
- **Passed:** Y
- **Warnings:** Z
- **Errors:** W

### Results

#### ✅ Passed
- [x] 结构完整性
- [x] 目标清晰度
- [x] 任务独立可验证

#### ⚠️ Warnings
- [ ] 任务粒度不均 (建议拆分 Task 2)
- [ ] 验证项不够具体

#### ❌ Errors
- [ ] 文件引用不存在: `src/missing.ts`

### Recommendations
1. [建议1]
2. [建议2]

### Verdict
- [ ] **APPROVED** - 计划可执行
- [ ] **NEEDS_REVISION** - 需要修订后重新检查
- [ ] **REJECTED** - 计划不可行，需重新规划
```
